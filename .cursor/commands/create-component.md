# Створення React-компонента

Ти допомагаєш **додати новий UI-компонент** у монорепі Excalidraw. Спочатку уточни або визнач за контекстом, **де** він живе.

## Де розмістити

- **`excalidraw-app/`** — специфіка веб-застосунку (сторінки, обгортки, app-only UI).
- **`packages/excalidraw/`** — бібліотека редактора, що публікується як `@excalidraw/excalidraw`; зміни тут впливають на споживачів пакета.
- Інші пакети (`packages/common`, `element`, `utils` тощо) — лише якщо компонент справді низькорівневий/спільний; не дублюй те, що вже є.

## Обов’язкові конвенції (`.cursor/rules/conventions.mdc`)

- **Функціональний компонент** + hooks (без class components).
- Файл компонента: **PascalCase**, наприклад `MyWidget.tsx`.
- **Named export** тільки (без `default export`).
- Тип пропсів: **`MyWidgetProps`** (або узгоджено з сусідніми файлами в тій же папці).
- Утиліти в тій же зоні — **kebab-case** файли, наприклад `my-widget-helpers.ts`.
- TypeScript strict: без `any`, без `@ts-ignore`, типи для публічних пропсів; `import type { ... }` де доречно.

## Архітектура редактора (якщо компонент у `packages/excalidraw`)

- Стан редактора змінюється через **ActionManager** / існуючі шляхи оновлення сцени — не вводь Redux/Zustand для ядра.
- Не малюй сцену через React DOM — канвас залишається в pipeline рендеру (див. `.cursor/rules/architecture.mdc`).
- **Не змінюй** файли з `do-not-touch` без явного схвалення.

## Тести (`.cursor/rules/testing.mdc`)

- Колокація: **`MyWidget.test.tsx`** поруч із `MyWidget.tsx`, якщо для цього шару прийняті colocated tests.
- Використовуй існуючі test-utils з пакета/апки; уникай крихких снапшотів без потреби.

## Після створення

1. `yarn test:typecheck` з кореня репозиторію.
2. За потреби: `yarn test:app --watch=false` або вузький запуск Vitest по файлу тесту.
3. `yarn fix` — якщо ESLint/Prettier лаються на нові файли.

## Що зробити зараз

1. Якщо користувач не назвав шлях — запропонуй конкретний шлях (`excalidraw-app/...` або `packages/excalidraw/...`) і ім’я компонента.
2. Створи компонент і за потреби тест, імпорти та експорти з узгодженням із сусідніми файлами в тій же директорії.
3. Не розширюй scope: без рефакторингу не пов’язаних файлів і без нових npm-залежностей без узгодження.
